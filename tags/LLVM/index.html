LLVM Essentials阅读笔记

关于

这是笔者阅读《LLVM Essentials》这本书过程中的一些笔记存档，方便之后进行查阅。

Chapter 1 - Playing With LLVM

正如本章标题所言，作者在本章里向我们展示了LLVM的大致架构，以及我们应该如何快速使用LLVM提供的一些工具链。LLVM是伊利诺伊大学于2000年启动的一个研究项目，最初的目的是为高级编程语言提供基于SSA的编译优化技术，但LLVM的发展已经远不止于此，它已经成为了一个包含众多子项目的umbrella project。在本章中，作者向我们简单介绍了LLVM模块化的设计理念、LLVM IR以及一些常用工具链。

Modular design and collection of libraries

作者在以写一个LLVM Pass为例，在LLVM中，编译器开发者所编写的一个Pass可以被独立编译生成.o文件，最后生成一个.a的静态链接库。LLVM提供了LLVM PassManager的机制，使得其可以通过Pass之间的依赖信息来选择性地链接和执行需要的Pass。下面的图展示了这一机制：

除此之外，一些优化算法(Optimizer)，例如指令调度、寄存器分配等也可以被组织为独立的Pass来做选择性的执行。作者还提到，对于特定的target，除了写特定的Pass之外，还可以用tablegen的方式，编写描述性的.td文件。还有一些诊断工具，例如Bugpoint，其可以用来定位错误是出现在哪一个Pass中。

LLVM IR

LLVM IR是LLVM项目的核心，其不仅可以用于对接不同语言的前端，也可以用于中间代码优化和后端平台的代码生成。在LLVM中，LLVM IR有三种存在形式：

  • In-Memory IR：也就是我们在写Pass时所操作的IR的数据结构
  • On-disk bitcode representation
  • Human-readable form (LLVM Assembly)

在这作者给了一个简单的例子：

在这个C程序中，定义了一个全局变量globalvar以及一个函数add，add函数接收一个整形参数，返回全局变量和该整形参数相加的值。
接着可以用下面的命令，将上面的add.c程序转换成其对应的LLVM IR
clang -emit-llvm -c -S add.c
下面是对这一组命令行参数在clang中解释以及在LLVM 17中的运行结果：

  • -emit-llvm：Use the LLVM representation for assembler and object files
  • -c：Only run preprocess, compile, and assemble steps
  • -S：Only run preprocess and compilation steps

从命令行参数的含义可以看出，这里得到的add.ll在LLVM中所处的层次是LLVM Assemble，它包含了人类可读的LLVM IR，LLVM 17下的结果和书中基本一致。
该文件主要包含以下几个部分：

  • ModuleID：源文件add.c对应了一个LLVM Module，LLVM Module是LLVM中的一个顶层数据结构，它包含了函数、全局变量、函数原型以及符号表项等信息。
  • Data Layout & Target Triple：第一个e表示little endian，m-e表示elf类型，之后描述了一些类型以及各自占多少字节。
  • 全局变量和局部变量：在LLVM IR中，全局变量通过@加上变量名的方式来描述，并且将其视作指针；局部变量又分为寄存器变量以及通过alloca指令从stack frame分配出的栈变量
  • LLVM IR里的函数函数：描述了参数类型和返回值类型，计算过程和C比较类似
  • Attributes：和LLVM函数相关联的一组属性
  • 一些编译制导语句以及使用的变量的值：llvm.module.flags和llvm.ident

LLVM tools and using them in the command line

作者在这一节介绍了以下工具：

  • llvm-as：LLVM assembler，将LLVM IR转换成bitcode
  • llvm-dis：LLVM disassembler，将bitcode转换成LLVM IR
  • llvm-link：用于链接bitcode文件
  • llvm-lli：一个bitcode的解释器
  • llvm-llc：将bitcode转换为目标机的assembly
  • opt：LLVM Analyzer and Optimizer，通过opt [options] [input file name]的方式使用

    • 如果使用-analyze选项，常见的可用分析有：

      • basicaa：基础的别名分析
      • da：依赖分析
      • …

    • 如果不用-analyze选项，则会做一些优化，这些优化可以是用户自己写的Pass，或者一些内置的优化，例如

    • constprop：简单的常量传播

    • dce：死代码消除

    • …

通过介绍这些工具，整个LLVM从源代码到目标机binary的路径也更加明确了起来，大致会经过source code(clang -emit-llvm -c) -> LLVM IR(llvm-as) -> bitcode(llvm-llc) -> binary这样的路径，其中每两种不同的表示之间都存在相应的转换工具。
